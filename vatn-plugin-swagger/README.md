# vatn-plugin-swagger

Serves an OpenAPI 3.0 spec and a Swagger UI for exploring your node's API.

## How it works

Registers two HTTP endpoints: `GET /api-docs` returns an OpenAPI 3.0 JSON document, and `GET /docs` returns a static HTML page that loads Swagger UI from the jsDelivr CDN and points it at `/api-docs`. If no pre-built spec is supplied via `withSpec(...)`, `OpenApiBuilder` generates a minimal skeleton from the title, version, description, and servers list provided in `SwaggerConfig`. No service interface is registered.

## Setup

```xml
<dependency>
    <groupId>dev.vatn.plugins</groupId>
    <artifactId>vatn-plugin-swagger</artifactId>
    <version>1.0-alpha.12</version>
</dependency>
```

```java
// Generated skeleton spec
VNodeRunner.create()
    .addPlugin(new SwaggerPlugin(
        SwaggerConfig.of("My VATN API", "1.0.0")
            .withDescription("Internal API for my service")
            .withServers(List.of("https://api.example.com"))))
    .run();

// Pre-built spec
String openApiJson = Files.readString(Path.of("openapi.json"));
VNodeRunner.create()
    .addPlugin(new SwaggerPlugin(
        SwaggerConfig.of("My VATN API", "1.0.0")
            .withSpec(openApiJson)))
    .run();
```

## API

This plugin registers no service interface.

| Endpoint    | Method | Response                              |
|-------------|--------|---------------------------------------|
| `/api-docs` | GET    | OpenAPI 3.0 JSON (`application/json`) |
| `/docs`     | GET    | Swagger UI HTML page                  |

## Configuration

| Option        | Default     | Meaning                                                      |
|---------------|-------------|--------------------------------------------------------------|
| `title`       | ‚ÄĒ           | `info.title` in the OpenAPI spec                             |
| `version`     | ‚ÄĒ           | `info.version` in the OpenAPI spec                           |
| `description` | not set     | `info.description` in the OpenAPI spec                       |
| `spec`        | generated   | Pre-built OpenAPI 3.0 JSON string; bypasses `OpenApiBuilder` |
| `servers`     | not set     | List of server URLs added to the `servers` array             |

## Notes

- The generated skeleton from `OpenApiBuilder` contains only metadata ‚ÄĒ no paths or schemas. Supply a complete spec via `withSpec(...)` for a useful Swagger UI.
- The Swagger UI HTML loads JS and CSS from `cdn.jsdelivr.net`; the page will not render in environments without outbound internet access.
- Both endpoints are unauthenticated by default; combine with `vatn-plugin-cors` or a network-layer access control if the spec contains sensitive schema information.
- `withSpec(json)` accepts any valid OpenAPI 3.0 JSON string, including specs generated by tools like Springdoc or the OpenAPI Generator.
